# 02 - Modules and Features

This document defines the planned JDHub modules, their purposes, core functions, and future sub-functions.

The goal is to define the full direction without building everything at once.

## 1. Dashboard

### Purpose

The Dashboard is the first screen after login. It gives a quick overview of the most important information across JDHub.

### Core Functions

- Show today's summary
- Show recent life logs
- Show open tasks
- Show upcoming reminders
- Show upcoming schedule items
- Show finance snapshot
- Show active projects
- Show server status summary
- Show recent command actions

### Future Sub-Functions

- Custom dashboard widgets
- Drag-and-drop dashboard layout
- Daily summary generated by Command Center
- Alerts for overdue tasks, unpaid bills, or server issues

---

## 2. Command Center

### Purpose

The Command Center is the main input and search layer of JDHub.

Instead of only clicking buttons and forms, the user can type commands naturally.

### Core Functions

- Add life logs
- Create tasks
- Add reminders
- Add schedule items
- Log finance transactions
- Search existing records
- Summarize records
- Save command history
- Preview detected actions before saving

### Example Commands

```text
add note: blackout happened and nginx API was unhealthy
create task: remove public port 5678 priority high
log expense: 250 food lunch cash
schedule: gym tomorrow at 5pm
search: nginx blackout
summarize this week
```

### Future Sub-Functions

- Local AI command understanding
- Outside AI integration through safe backend tools
- Voice command input
- Suggested actions
- Multi-step command flows
- Ask questions across all modules

---

## 3. Life Log

### Purpose

Life Log stores personal records, events, thoughts, ideas, decisions, technical fixes, health notes, and incidents.

It is an upgraded journal, but not only for feelings or diary entries.

### Core Functions

- Create life log entry
- Edit entry
- Delete/archive entry
- Search entries
- Filter by category
- Filter by tags
- Attach files later
- Link entries to projects, tasks, server incidents, or finance items later

### Categories

- Personal
- Work
- Coding
- Server
- Health
- Money
- Idea
- Incident
- Learning

### Example Entry

```text
Blackout happened. Acer server became unreachable. Nginx Proxy Manager showed API unhealthy after reboot.
```

## 6. Scheduling

### Purpose

Scheduling stores events, planned time blocks, appointments, and personal schedule items.

This should start as an internal JDHub schedule first. Google Calendar integration should come later.

### Core Functions

- Create schedule item
- Add date/time
- Add location
- Add notes
- Link to project/task/reminder
- View daily/weekly schedule

### Example Schedule Items

- Gym
- Server maintenance
- Work session
- Appointment
- Project sprint session
- Bill payment date

### Future Sub-Functions

- Google Calendar integration
- Import calendar events
- Create/update Google Calendar events
- Conflict detection
- Recurring schedules
- Calendar view

---

## 7. Finance

### Purpose

Finance tracks personal money data: expenses, income, budgets, bills, and savings goals.

### Core Functions

- Finance Overview
- Budget categories
- Transactions
- Bills
- Savings Goals
- Basic monthly summary

### Finance Sections

#### Overview

Shows income, expenses, remaining budget, unpaid bills, and savings progress.

#### Budget

Stores spending limits per category.

#### Transactions

Stores income and expenses.

#### Bills

Stores recurring and one-time bills.

#### Savings Goals

Stores financial goals like emergency fund, UPS, PC upgrade, or server upgrade.

### Future Sub-Functions

- Receipt linking
- OCR receipt extraction
- CSV import
- Charts
- AI spending summary
- SQL-backed finance storage if stricter structure is needed

## 11. Receipts

### Purpose

Receipts capture receipt images and receipt-related data.

Receipts should later connect to Finance transactions.

### Core Functions

- Add receipt manually
- Upload receipt image later
- Store merchant, amount, date, category, and notes
- Link receipt to finance transaction

### Future Sub-Functions

- OCR extraction
- Discord receipt bot
- Messenger receipt bot
- Automatic transaction creation
- Receipt search

---

## 12. Automations

### Purpose

Automations stores information about workflows in tools like n8n, Make.com, Discord bots, and future internal automation flows.

### Core Functions

- Add automation record
- Store platform
- Store purpose
- Store webhook URL or reference
- Store status
- Link automation to project

### Future Sub-Functions

- Trigger webhook manually
- View last run status
- n8n API integration
- Make.com reference records
- Automation error logs

---

## 13. Server Manager

### Purpose

Server Manager tracks personal infrastructure like the Acer server, Docker containers, Nginx Proxy Manager, exposed ports, and incidents.

### Core Functions

- Add server notes
- Add server incident logs
- Track Docker containers manually first
- Track ports manually first
- Track domains/subdomains manually first

### Future Sub-Functions

- Live Docker status
- Disk/RAM/CPU metrics
- Nginx Proxy Manager checks
- Uptime monitoring
- SSH command execution with confirmation
- Public port exposure alerts

---

## 14. Integrations

### Purpose

Integrations stores future external connections.

### Planned Integrations

- Google Calendar
- Google Drive
- GitHub
- Codex/repository references
- n8n
- Make.com
- Discord
- Telegram
- Email
- Local AI/Ollama
- Outside AI provider

### Core Rule

Integrations should not be required for the MVP.

The core JDHub modules should work first. Integrations should enhance the modules later.
